CSS for Twitch Extensions
A tiered approach to build theme-able and reusable CSS
The 3-tier SASS framework
The SASS files in this framework are organized into tiers by levels of reusability. The SASS files can then be compiled into separate CSS files for each Twitch extension type, e.g. panel.css, video_overlay.css, mobile.css and config.css.
  1. The first tier of the SASS files is built on top of the utility classes found in BassCSS, with some common components added like buttons and popovers, also basic animated transitions.
  2. The second tier of the SASS files defines the theme of the extension, e.g. a League of Legends game theme. In the example files we are using a theme specific prefix for the SASS file and the classnames that are not reusable for other themes - "mygame-" for a blue sample theme and "lol-" for League of Legends game theme. Files in this tier may use semantic class names sometimes when the stylings are not reusable for extensions of a different theme. For example, .lol-winrate is used to define the styling of the League of Legends player's win rate in the game.
  3. The third tier of the SASS files defines additional CSS classes that are only used by a specific extension type (panel, video overlay and mobile), and the config pages (extension config and extension live config). This is where the base size of rem is specified so the base size can vary according to the dynamic size of a specific extension type. Same with the files in the 2nd tier, we use theme specific prefix, e.g. "lol-" for League of Legends game theme. Semantic class names may be used in this tier too.
This framework can also be used for non Twitch-extension web sites. Simply exclude the third tier and compile only the SASS files under the first and second tiers.
How to use it for your Twitch extension project
You can use the already compiled CSS files and start building the layout. If you wish to create your own theme, first have a look at how the styling is defined in each of the 3 tiers of SASS files. You can simply change the color values and fonts, modify components like buttons or animation, or go further to add more classes.
